#### 🗄️ Backend Layer & Middleware Architecture (`server.js`)

The backend behaves as a **Zero-Dependency High-Velocity Hub Router**. It completely avoids external database handshake constraints by utilizing localized memory variables.

* **Middleware Execution Scheme:** * `cors()` intercepts incoming network payloads, ensuring that your React Client running on Port `3000` is permitted to securely transmit network data packets directly to Port `5000` without triggering browser cross-origin policy exceptions.
* `express.json({ limit: '50mb' })` sets an elevated buffer size limit. This is explicitly configured to ensure that when a Farmer uploads raw image streams, the large Base64 text blobs are parsed directly without causing stream truncation or dropping transaction parameters.


* **Local Memory Storage Shards:** Data rows for users (`LOCAL_USER_DATABASE`) and harvest batches (`LOCAL_LISTINGS_DATABASE`) reside within the system's rapid active RAM variables. This design provides near-zero query times and isolates your system from external cloud availability drops or OpenSSL security handshake errors.

#### 🐍 Data Crawler, Cache, & AI Microservice Architecture (`scraper.py` / `app.py`)

This element functions as an isolated background task manager and a deep learning regression processor.

* **Multi-Threaded Thread Guarding:** The crawler utilizes an isolated background thread (`threading.Thread`) that executes a continuous loop, waking up every 5 minutes to scrape agricultural data from the National Agmarknet Portal without blocking your main application endpoints.
* **Thread Safety (`Lock` Mutex):** Because the background thread overwrites active prices while the user thread continuously reads them, a thread synchronization lock (`CACHE_MUTEX = threading.Lock()`) coordinates memory access. This prevents race conditions and data corruption across system scopes.
* **O(1) Hash-Map Cache Efficiency:** The system cleans crop text inputs into normalized keys (e.g., `"Paddy (Common)"` becomes `"paddycommon"`). This enables the lookup loop to query the dictionary in constant $O(1)$ time complexity, rather than performing an $O(N)$ linear array scan, dropping database stress to zero.
* **Forward-Pass AI Regression Curve (PyTorch):** When a user triggers price forecasting, inputs are converted into a PyTorch floating-point matrix tensor `[base_price, interval_day, crop_name_length]`. This tensor passes directly through a custom neural network module block (`nn.Module`) to predict pricing variances for the upcoming week.

#### 💻 Interface UI Layer, State Machine, & Styles (`App.js`)

The React application acts as an optimized, responsive state coordinator.

* **Dynamic Role-Based Workspaces:** The portal checks the authenticated active user state configuration (`user.role`). It dynamically changes available navigation buttons and viewport structures depending on whether a **Farmer** needs to list a harvest batch, a **Merchant** needs to inspect crop cards, or an **Admin** needs to manage accounts.
* **Debounced Input Performance Optimization:** The search entry field features an automated network throttling mechanism (`searchDebounceRef`). When a user types continuously (e.g., typing *"Wheat"* quickly), the system clears previous evaluation timers and delays the outgoing API fetch call until typing pauses for 250ms. This prevents the application from making redundant network requests on every keystroke.
* **Enterprise Glassmorphism Style Scheme:** Visual structures rely on a clean UI overlay layout. It uses structural translucent backdrop property blurs (`backdrop-filter: blur()`), glowing background orbs, and distinct visual hierarchies to maintain a modern, professional appearance across varied screen sizes.
